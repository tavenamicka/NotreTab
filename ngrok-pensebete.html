Pense-bête ngrok — NotreTab

Exposer l'application en externe pour tests mobiles · Avec Basic Auth

Utilisation (chaque session)
1
Lancer l'application

Terminal 1 — dans le dossier du projet :

npm run dev

Attendre que Vite affiche "ready" sur le port 5173

2
Lancer le tunnel ngrok

Terminal 2 — remplacer le mot de passe par le tien :

ngrok http 5173 --basic-auth="mickael:TonMotDePasse"
3
Récupérer l'URL publique

ngrok affiche une ligne Forwarding :

Forwarding https://abc123xyz.ngrok-free.app → http://localhost:5173

Cette URL est unique à chaque lancement. À envoyer sur ton téléphone.

4
Ouvrir sur mobile

Saisir l'URL dans le navigateur → une fenêtre demande :

Identifiant : mickael Mot de passe : TonMotDePasse
5
Arrêter le tunnel

Dans le Terminal 2 :

Ctrl + C

L'URL cesse immédiatement d'être accessible depuis l'extérieur.

Commandes utiles
CommandeUsage
ngrok versionVérifier l'installation
ngrok http 5173
--basic-auth="user:pwd"		 Tunnel avec auth

🔧 Configuration (une seule fois)

Associer ngrok à ton compte :
ngrok config add-authtoken TON_TOKEN
Token disponible sur dashboard.ngrok.com

Surveiller les requêtes (inspection)
📡 Interface locale

Pendant que ngrok tourne, ouvrir dans le navigateur du PC :

http://127.0.0.1:4040

Tu vois en temps réel :
· Toutes les requêtes entrantes
· L'IP source de chaque appel
· Le détail des headers / body
· Un bouton Replay pour rejouer

Règles de sécurité
⛔ À ne jamais faire

· Laisser ngrok ouvert sans surveillance
· Partager l'URL + identifiants publiquement
· Utiliser en production avec de vraies données
· Oublier d'arrêter le tunnel après les tests

✅ Bonnes pratiques

· Changer le mot de passe Basic Auth régulièrement
· Surveiller l'interface 127.0.0.1:4040 pendant les tests
· Fermer avec Ctrl+C dès la fin de session
· Utiliser uniquement pour tests avec données fictives
